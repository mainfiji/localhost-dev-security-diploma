# Инцидент: Фишинговая атака на почту m.romanov@local.host

## Обнаружение

- **Источник:** Анонс публикации переписки в даркнете
- **Пользователь:** Романов Михаил (IT-отдел)
- **Действие:** Переход по ссылке из письма, ввод пароля на поддельной странице

Чувствительная информация
Злоумышленники получили доступ к переписке, содержащей:

Внутренние регламенты и политики

Данные о сотрудниках

Информацию о проектах

Учётные данные для доступа к системам

Тип атаки
Фишинговая атака (credential harvesting)

Механика:

Пользователь получает письмо с вредоносной ссылкой

Переходит на поддельный сайт (mail.evil.com)

Вводит учётные данные

Данные отправляются злоумышленнику

Цель: получение доступа к почтовому ящику и дальнейший доступ к инфраструктуре.

Устранение
```bash
# Смена пароля
passwd m.romanov

# Сброс всех сессий
loginctl terminate-user m.romanov

# Блокировка IP злоумышленника
nft add rule inet filter forward ip saddr 185.130.5.44 drop
Настройка антифишинга
bash
# SPF
v=spf1 mx include:local.host ~all

# DKIM
# Создание ключей и настройка DNS

# DMARC
_dmarc IN TXT "v=DMARC1; p=reject; rua=mailto:dmarc@local.host"
```
Корневые причины
Недостаточная осведомлённость сотрудников о фишинге

Отсутствие фильтрации фишинговых писем

Отсутствие 2FA для почтовых аккаунтов

Отсутствие SPF/DKIM/DMARC

Предотвращение
Регулярные тренировки по распознаванию фишинга

Внедрение почтового шлюза с антифишинговыми фильтрами

Включение 2FA для всех почтовых аккаунтов

Настройка SPF/DKIM/DMARC

Мониторинг подозрительных входов (Wazuh)
