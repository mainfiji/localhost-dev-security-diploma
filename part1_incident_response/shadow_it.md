# Задача 4. Поиск теневых объектов

## Цель

Провести сканирование корпоративной сети для поиска неучтённых и посторонних объектов.

## Методология

| Параметр | Значение |
|----------|----------|
| Хост для сканирования | srv_ansible (192.168.205.77) |
| Причина выбора | Доступ ко всем подсетям, настроены инструменты администрирования |
| Команда | `sudo nmap -sS 192.168.201.0/24 192.168.202.0/24 192.168.203.0/24 192.168.204.0/24 192.168.205.0/24 192.168.206.0/24 10.97.23.0/24` |

## Обнаруженные теневые объекты

### Хост 1: 192.168.205.88

| Параметр | Значение |
|----------|----------|
| ОС | Linux (ядро 4.15–5.8) |
| Сегмент | admin_net (192.168.205.0/24) |
| Открытые порты | 22/tcp (SSH), 3389/tcp (RDP/xrdp), 8081/tcp (HTTP) |
| Прикладное ПО | phpLDAPadmin 1.2.5 |
| Предполагаемое назначение | Сервер администрирования LDAP (не учтён в схеме) |

### Хост 2: 192.168.202.99

| Параметр | Значение |
|----------|----------|
| ОС | Linux (OpenSSH 9.6p1) |
| Сегмент | users_net (192.168.202.0/24) |
| Открытые порты | 22/tcp (SSH), 3389/tcp (RDP) |
| Предполагаемое назначение | Рабочая станция пользователя (не учтена в схеме) |

## Выводы

Обнаружены 2 неучтённых хоста:
1. Сервер phpLDAPadmin в сегменте управления — потенциальный риск компрометации LDAP
2. Пользовательская ВМ в сегменте пользователей — нарушение учёта активов

## Рекомендации

1. Внести хосты в учёт активов (инвентарь Ansible, схема сети)
2. Для phpLDAPadmin: настроить аутентификацию и ограничить доступ по IP
3. Проверить легитимность пользовательской ВМ (возможно, это скомпрометированный хост)
