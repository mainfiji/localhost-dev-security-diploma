# Инцидент: Утечка данных из веб-приложения

## Обнаружение

- База данных клиентов опубликована в открытом доступе
- Данные: ФИО, адреса, телефоны клиентов
- Сайт: local.host (WordPress + API на порту 8080)

## Выявленные недостатки

| Недостаток | Описание |
|------------|----------|
| Открытый порт 8080 | Тестовый API доступен извне без аутентификации |
| Уязвимый плагин | Pie Register 3.7.1.4 (CVE-2025-34077) |
| Обход директорий | Доступ к /wp-admin/ через уязвимость |
| API без аутентификации | `/api/v1/customers` возвращает JSON с персональными данными |
| Информация о версии | `/wp-content/plugins/pie-register/readme.txt` |
| phpinfo | `/phpinfo.php` доступен извне |
| robots.txt | Раскрывает /wp-admin/ |

## Уязвимости

### CVE-2025-34077 — Authentication Bypass (критическая)

**Шаги воспроизведения:**
1. POST-запрос на `/wp-admin/admin-ajax.php`
2. Параметры: `action=pie_login_social social_site=true user_id_social_site=1`
3. Сервер возвращает сессионную cookie для пользователя с ID=1
4. Подстановка cookie → доступ к `/wp-admin/`

## Цепочка атаки

1. **Разведка** → Обнаружение уязвимых эндпоинтов через Dirb
2. **Обход аутентификации** → CVE-2025-34077 (получение admin-доступа)
3. **SQL-инъекция** → Через API `/api/users?id=1' OR '1'='1`
4. **Извлечение данных** → UNION SELECT username, password FROM users
5. **Доступ к CRM** → Через административную панель
6. **Эксфильтрация** → Выгрузка базы данных

## Пароль администратора

**Пароль:** `admin` (подобран после получения хеша)

## Корневые причины

1. Недостаточное тестирование безопасности перед релизом
2. Отсутствие WAF на сайте
3. API доступен без аутентификации
4. Устаревший плагин с известной уязвимостью

## Предотвращение

1. Внедрение безопасного SDLC (DevSecOps)
2. Обязательный аудит кода перед релизом
3. Внедрение WAF (BunkerWeb)
4. Политика сложных паролей и 2FA
5. Регулярное сканирование уязвимостей (Wazuh Vulnerability Detector)
6. Закрытие API за VPN
