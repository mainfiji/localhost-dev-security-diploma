# Задача 6. Анализ привилегированного доступа

## Организация доступа

- Права локального администратора на рабочих станциях предоставляются через группу `SG-ADM-T2-WKS-LOCALADMINS`
- В `/etc/sudoers.d/ldap-sudo` указано: `%SG-ADM-T2-WKS-LOCALADMINS ALL=(ALL:ALL) ALL`
- Ограничение входа через `/etc/security/access.conf` — разрешены группы `SG-ADM-T2-WKS-LOCALADMINS` и `SG-PC-USERS`

## Состав группы SG-ADM-T2-WKS-LOCALADMINS

| Учётная запись | Должность | Соответствие |
|----------------|-----------|--------------|
| adm_msvetlov | Начальник IT |  Соответствует |
| adm_mromanov | Ведущий специалист IT |  Соответствует |
| adm_ajukov | Инженер IT |  Соответствует |
| adm_rsafin | Специалист IT |  **Избыточно** |
| adm_student | Учебная |  Требуется проверка |

## Избыточные полномочия

**adm_rsafin** включён в группу `SG-ADM-T0-DIRECTORY-ADMINS` (администраторы LDAP), хотя его основные обязанности — поддержка файловых серверов и пользовательских хостов.

Это нарушает принцип минимальных привилегий (ПУД 3.2.1).

## Предлагаемые меры

1. Удалить `adm_rsafin` из группы `SG-ADM-T0-DIRECTORY-ADMINS`
2. Оставить только группы:
   - `SG-ADM-T1-SRV-SAMBA-ADMINS`
   - `SG-ADM-T2-WKS-LOCALADMINS`
3. Провести ревизию всех административных групп на сервере LDAP
4. Настроить регулярный аудит состава групп (ежеквартально)
