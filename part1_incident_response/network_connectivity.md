# Задача 1. Восстановление сетевой связности офисов

## Описание проблемы

Сотрудники офиса в Санкт-Петербурге не могут войти в систему. Руководство предположило, что проблема связана с переключением головного офиса на нового интернет-провайдера.

## Исследование

### Офис МСК (центральный)
- Подключение к бастиону МСК по RDP (порт 43389)
- Проверка VPN-шлюза `http://srv_wg_portal:8888`
- **Обнаружено:** пир отключён

### Офис СПБ
- Подключение к бастиону СПБ по RDP (порт 43389)
- Проверка МСЭ `branch_fw` через `docker exec`
- Проверка VPN-шлюза `http://srv_branch_wg_gate:8888`
- **Обнаружено:** указан неверный IP-адрес Endpoint (старый провайдер)

## Выявленные проблемы

1. **Центральный офис (МСК):** пир VPN отключён (возможно, после сбоя)
2. **Офис СПБ:** в конфигурации WireGuard указан старый IP-адрес провайдера

## Решение

1. Включен пир на `srv_wg_portal`
2. Обновлён Endpoint в конфигурации WireGuard на `srv_branch_wg_gate`:
   ```bash
   wg set wg0 peer <публичный_ключ> endpoint <новый_IP_провайдера>:51820
Проверена связность:

bash
ping 192.168.201.199  # LDAP-сервер центрального офиса
Результат
Связность между офисами восстановлена. Сотрудники СПБ могут входить в систему через LDAP-аутентификацию.
