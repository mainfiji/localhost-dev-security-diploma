## Ключевые результаты

### 1. Анализ системных проблем

Выявлены 8 системных проблем:

| № | Проблема | Пути решения |
|---|----------|--------------|
| 1 | Нарушение RBAC (прямое добавление в ресурсные группы) | Внедрение строгой модели SG-ORG → SG-RES |
| 2 | Слабая парольная политика в LDAP | Внедрение ppolicy с minlen=8, maxage=180d, lockout=5 |
| 3 | Отсутствие сегментации сети | Создание 5 сегментов для СПБ |
| 4 | Отсутствие egress filtering | Настройка запрета исходящего трафика, кроме разрешённого |
| 5 | Отсутствие антивирусной защиты | Внедрение ClamAV через Ansible |
| 6 | Недостаточное логирование | Подключение Samba audit + Wazuh |
| 7 | Уязвимости веб-приложений (SQLi) | Внедрение WAF, prepared statements |
| 8 | Отсутствие обучения сотрудников | Фишинговые тренировки, MFA |

### 2. Доработка корпоративных стандартов

- **Политика управления учётными записями и доступом** — добавлены разделы: требования к паролям, модель RBAC, управление привилегированным доступом, MFA, аудит.
- **Инструкция по безопасной настройке Linux** — добавлены: ppolicy, auditd, запрет запуска из /tmp, требования к веб-приложениям.
- **Создан новый стандарт для ОС Windows** — парольная политика, ограничение доступа по RDP, аудит, GPO.

### 3. Устранение отклонений от требований регулятора

Настроены:
- pwquality.conf, login.defs, faillock.conf на всех ws_msk*
- ppolicy на srv_ldap (check-password-module.ldif, policy-users.ldif, policy-service.ldif)
- SSH (PermitRootLogin no, LogLevel INFO)
- rsyslog с ротацией 52 недели
- Синхронизация времени с srv_ntp

### 5. Антивирусная защита

- Разработан Ansible-плейбук `clamav.yml`
- Установлен ClamAV на 11 хостах + файловый сервер
- Настроено ежедневное сканирование через cron
- Проверка с EICAR-файлом пройдена успешно
- Результаты теста `test_a_result.txt` — все хосты OK

### 6. Active Directory

- Домен: `int.local.host`
- Созданы OU: Disabled Accounts, Groups (ADMIN, INFRA, ORG, RES), Users, Service Accounts
- Настроены GPO: парольная политика, блокировка, аудит, ограничение RDP
- Созданы сетевые потоки для AD (Kerberos, LDAP, SMB, DNS)
