# Групповые политики (GPO)

## 1. Политика паролей (Password Policy)

Настройки применены через GPO на уровне домена.

| Параметр | Значение |
|----------|----------|
| Минимальная длина пароля | 8 символов |
| Сложность пароля | Буквы разных регистров, цифры, спецсимволы |
| Срок действия пароля | 180 дней |
| История паролей (запоминать) | 3 последних пароля |
| Блокировка учётной записи после | 5 неудачных попыток |
| Сброс блокировки через | 15 минут |

---

## 2. Политика доступа (User Rights Assignment)

| Настройка | Значение |
|-----------|----------|
| Гостевая учётная запись | Отключена |
| Локальный пользователь Guest | Отключён |
| Интерактивный вход для локальных пользователей | Запрещён (кроме членов группы локальных администраторов) |
| Доступ по RDP | Разрешён только для групп `Domain Admins` и `RDP_Users` |

---

## 3. Политика аудита (Audit Policy)

Включены следующие категории событий (успешные и неудачные попытки):

| Категория | Подкатегория | Настройка |
|-----------|--------------|-----------|
| Logon/Logoff | Logon | Success, Failure |
| Logon/Logoff | Logoff | Success |
| Account Management | User Account Management | Success, Failure |
| Account Management | Security Group Management | Success, Failure |
| Policy Change | Audit Policy Change | Success, Failure |
| Object Access | File System | Success, Failure (для выбранной общей папки) |
