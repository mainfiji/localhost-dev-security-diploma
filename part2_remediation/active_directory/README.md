# Настройка Active Directory (тестовый стенд)

## Обзор

В компании планируется открытие нового офиса за границей, где будут использоваться ОС Windows.  
В рамках дипломной работы развёрнут тестовый стенд Active Directory для демонстрации возможностей централизованного управления пользователями и политиками.

## Инфраструктура

- **Домен:** `int.local.host`
- **Контроллер домена:** Windows Server 2022
- **Клиент:** Windows 10 (присоединён к домену)

## Содержание

- [Структура организационных единиц (OU)](ou_structure.md)
- [Групповые политики (GPO)](gpo_settings.md)
- [Сетевые потоки для Active Directory](network_flows.md)

## Проверка применения политик

- Клиентская ВМ (Windows 10) успешно присоединена к домену `int.local.host`.
- При входе под доменным пользователем применяются все настроенные политики.
- Локальные настройки на клиенте (например, парольная политика) заблокированы для редактирования – это подтверждает, что доменные политики применяются корректно.
- Доступ по RDP для пользователей из группы `RDP_Users` работает, для остальных – запрещён.

## Выводы

- Active Directory развёрнута и готова к эксплуатации в тестовом режиме.
- GPO настроены в соответствии с Политикой управления учётными записями и Инструкцией по настройке безопасности ОС Windows.
- Сетевые потоки документированы и могут быть использованы для настройки межсетевого экрана.

Все задачи по Active Directory выполнены.
