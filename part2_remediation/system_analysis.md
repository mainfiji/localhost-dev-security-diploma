# Анализ системных проблем ИБ

## Перечень системных проблем и возможные пути решения

| № | Описание системной проблемы | Источники (причины) | Пути решения |
|---|----------------------------|----------------------|--------------|
| 1 | Отсутствие централизованного контроля доступа и нарушение RBAC | – Заявки на доступ (Сафин получил T0 доступы без оснований)<br>– Файловый сервер (srv_branch_fs) изначально с guest ok = yes<br>– LDAP используется частично | – Внедрить строгую RBAC‑модель через LDAP<br>– Запретить прямое добавление пользователей в ресурсные группы<br>– Оставить только цепочку: SG‑ORG → SG‑RES<br>– Убрать guest‑доступ в Samba (guest ok = no, map to guest = never) |
| 2 | Слабая парольная политика в LDAP | – Подбор паролей (≥15 слабых паролей найдено) | – Внедрить ppolicy: минимум 8–12 символов, сложность, история паролей, блокировка после 5 попыток |
| 3 | Отсутствие сегментации сети | – Все хосты в одной сети (до переделки СПБ) | – Сегментация: branch_users_net, branch_servers_net, branch_admin_net, branch_dmz_net<br>– Firewall default deny: policy drop |
| 4 | Отсутствие фильтрации исходящего трафика (egress control) | – Майнер на srv_wiki подключался к moneroocean | – На МСЭ запретить всё исходящее, разрешить только: DNS, HTTP/HTTPS через прокси |
| 5 | Отсутствие антивирусной защиты | – ClamAV отсутствовал | – Внедрить ClamAV, cron‑сканирование, проверка EICAR |
| 6 | Недостаточное логирование и мониторинг | – Wazuh не видит srv_wiki<br>– нет логов Samba изначально | – Подключить все хосты к Wazuh<br>– Включить аудит: входы, доступ к файлам, Samba (full_audit) |
| 7 | Уязвимости веб‑приложений (SQLi) | – Нет: валидации input, backend‑фильтрации | – Использовать prepared statements, внедрить WAF, backend validation |
| 8 | Отсутствие обучения пользователей (фишинг) | – Пользователь ввёл пароль на фишинговом сайте | – Обучение пользователей, внедрение MFA, антифишинг фильтров |

## Корневые причины инцидентов (связь с требованиями)

| № | Описание инцидента / проблемы | Возможная корневая причина |
|---|-------------------------------|----------------------------|
| 1 | srv_wiki (майнер) | Отсутствие ЗВП.1 + КЦ.1 |
| 2 | Фишинг (почта) | Отсутствие ИАФ.2 + обучения |
| 3 | SQL‑инъекция | Отсутствие контроля входных данных (КЦ.1) |
| 4 | Слабые пароли LDAP | Отсутствие ИАФ.2 (ppolicy) |
| 5 | Неучтённые хосты | Отсутствие учёта активов и РСБ.1 |
